John directed the Alt1 approach: real Postgres columns added as needed per tenant (schema-per-tenant), with am_field_spec surviving as the column registry. His verbatim direction: standard per-tenant tables for canonical facts plus a per-tenant custom ornament where columns are added as needed.

The Alt1 design named "Full Ornament — One Fact, One Column" was produced and proposed. Key decisions embedded in the design:

- A dedicated 1:1 ornament table (ai_venture_ext) is used rather than widening ai_venture, to contain ALTER lock blast radius and preserve a separate 1,600-column budget.
- Only project_property scalar fields move to real columns. Form/WO field values stay in ai_field_entry because the grain is per-instance (4.9% of venture+spec pairs have multiple entry rows, max 7).
- FILE property fields (384 of 695 BRI0 property specs) stay in ai_attachment and are not columnarized.
- task_virtual mirrors (3,235 specs) remain in EAV in V1; retiring them is deferred to an optional Phase 6.
- All DDL goes through a single chokepoint, lib/ornament.js, and never by hand or migration for custom columns.
- Column names are minted immutable identifiers (slug + spec_id), never derived from user labels, so rename cannot disable consumers.
- Lazy column minting is used for sync/import-minted specs (column created on first non-null value, not on declaration), keeping worst-case tenant at approximately 680 columns instead of 1,122.
- A write-protection trigger on ai_venture_ext enforces single-writer for workflow-authority columns via a GUC guard; rogue writes raise an error rather than silently corrupting.
- Workflow-fact columns (installation_start_date class) are declared by-ID in a settings screen, written only through the lib/task-virtual-entry.js chokepoint, and are nightly-recomputable from ai_task_instance via reconcile.js.
- The nightly reconciler diffs am_field_spec (registry desired state) against information_schema.columns across all ~20 AIEPC schemas and repairs with ADD COLUMN IF NOT EXISTS; a heartbeat row is asserted by /healthz to prevent silent reconciler death.
- A six-phase migration plan (P0–P5, with P6 optional) was defined, each phase independently shippable with a named rollback path. P3 is the explicit one-way commitment point (entry deletion after archive dump).
- The Agent API (routes/agent-api/*) adopts the design-3 API contract: GET /api/v1/projects/:id/facts keyed by column_name, PATCH with 409 machine_written for workflow columns and If-Match stale_write guard.
- A CI grep-gate test will fail the build on new occurrences of label ILIKE, def_group_label IS DISTINCT FROM, or title ILIKE in routes/lib.
