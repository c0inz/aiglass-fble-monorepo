## Decisions made

The architect (AI agent responding to the owner's brief) produced a complete, opinionated architecture called **The Fact Harness**. The following were decided within the session as part of the design deliverable:

- The label/title/phase-name inference model is to be replaced by a declared binding layer; capture tables are not rebuilt.
- Three new tables are the implementation target: `am_fact_def` (registry), `am_fact_binding` (harness), `ai_fact_value` (ledger), to be added as migration 163 in three places (BRI0, MAS0, `lib/tenant-provisioner.js`).
- Bindings are by `am_field_spec.id`, not by label or `internal_name`; this removes the `am_field_spec` dedup-merge from the critical path.
- The shipped catalog will contain approximately 25 system facts covering Standard-panel columns, solar-EPC lifecycle dates/events, and money disambiguation.
- `project_value` is resolved by declaring `column:monetary_value` at priority 10 and `column:total_price` at priority 11 — giving the five money columns a declared winner for the first time.
- Survivorship is per-attribute: priority ascending, then recency; manual override is implicit priority 0 and never overwritten by promotion.
- The Promoter uses a recompute-the-cell strategy (not incremental fold) so write-through and backfill cannot disagree.
- A transform DSL is explicitly rejected; the only allowed coercions are `numeric_parse` and `date_parse`.
- `PATCH /api/v1/projects/:id/facts` is designated the API's first write endpoint; it writes a manual override only, never mutating capture.
- The v2 `fields[]`/`describeShape` data-shape contract is unblocked and continues as a separate track beneath the Fact Harness.
- Blueprint version-clone code must copy active bindings to successor specs; this is identified as the one real coupling and must land with Phase 2.
- The nightly reconciliation script (recompute-from-capture, diff, flag STALE) is a required part of Phase 2.
- A `grep`-in-CI single-writer discipline is prescribed: only `lib/facts/promote.js` may write to `ai_fact_value`.
