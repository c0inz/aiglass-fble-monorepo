## Decisions made

The operator decided to fix bug B5 (inbox search input retaining typed text on browser back/forward navigation) with a minimal, scoped change to `views/inbox.ejs` only. The root cause was confirmed as browser bfcache (back/forward cache) restoring the live DOM snapshot without a server round-trip. The fix chosen was a `pageshow` event listener that clears `input[name="search"]` only when `event.persisted` is true, preserving server-rendered values on normal first loads. `autocomplete="off"` was also added to the search input as belt-and-suspenders.

The heavier alternative — setting `Cache-Control: no-store` on the `/inbox` response in `server.js` — was explicitly considered and rejected because it would disable bfcache entirely for the page and harm Back/Forward navigation performance. Only `views/inbox.ejs` was touched. The fix was verified via `vm.Script` parse check.
