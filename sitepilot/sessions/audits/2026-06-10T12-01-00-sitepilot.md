## Decisions made

The agent (acting as senior architect) produced the "WAINSCOT" design: the named Alt1 architecture for SitePilot's custom field storage. The design was delivered as a complete document in response to John's direction to use real Postgres columns ("ornament") rather than the previously proposed Fact Harness projection (Plan v1, which John tabled).

The following specific decisions are embedded in the design delivered during the session:

- `project_property` scalar values (311 specs on BRI0) will move to real typed columns on a new 1:1 ornament table (`ai_venture_ext`). EAV rows for those fields will be deleted at Phase 3, after a verified dump and explicit owner approval.
- `form_field` and `task_virtual` values stay as EAV rows permanently; instance-grain prevents column representation.
- `FILE`-kind property values stay in `ai_attachment`; no column.
- A single DDL chokepoint `lib/ornament.js#ensureColumn(specId)` is the only permitted caller of `ALTER TABLE ai_venture_ext`. All other code is prohibited from issuing that DDL.
- Column names are minted as `p_<slug(label,≤40)>_<spec.id>` — immutable forever; a label rename never touches DDL.
- A reject-trigger (`trg_reject_column_backed_entry`) is installed at Phase 3 on `ai_field_entry` to make any leftover EAV writer for a column-backed spec fail loudly rather than fork truth.
- Promoted-fact columns (`f_<fact_key>` on the ornament) are limited to ~6–12 declared facts (installation_start_date class); they accept no manual/API/UI writes ever — recomputable-only.
- `am_field_spec` survives as the column registry. Its `column_name` and `storage` columns (added by migration 163) bind specs to physical columns.
- `am_promoted_fact` table is introduced for by-ID declared fact bindings with collapse rules; no label anywhere in the binding.
- The 612 BRI0 property specs currently lacking `internal_name` will be minted in Phase 0.
- Lazy column minting (add on first non-NULL write, not at spec declaration, for import/sync paths) keeps worst-case column count at ~680 rather than ~1,135.
- A nightly `scripts/ornament-reconcile.js` diffs registry against `information_schema.columns` across all ~20 AIEPC schemas and repairs with `ADD COLUMN IF NOT EXISTS`; the heartbeat surfaces in the daily digest.
- V1 phasing is defined as six phases (0 Identity, 1 Dual-home backfill, 2 Read cutover, 3 One home, 4 Promoted facts, 5 Fleet), each independently shippable with defined rollback.
- The Agent API surface exposes properties by `internal_name` key and facts by `fact_key`; `PATCH /facts/*` returns 405 directing to the capture source.
- `docs/ORNAMENT_ARCHITECTURE.md` will be the canonical reference; `EAV_ARCHITECTURE.md` will be updated with a banner noting properties moved and forms unchanged.
